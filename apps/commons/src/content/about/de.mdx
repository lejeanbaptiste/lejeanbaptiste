# Über LEAF

LEAF-Writer-Commons ist Teil der [LEAF Commons Tool Suite](https://www.leaf-vre.org/docs/about-leaf/leaf-commons), einer Sammlung webbasierter, modularer und benutzerfreundlicher Tools. Diese unterstützen Textkodierung, Named Entity Recognition, Web-Annotation, Textanalyse und Veröffentlichung, ohne dass Anwender Auszeichnungssprachen erlernen müssen. Zudem ermöglichen sie einen einfachen Wechsel zwischen interoperablen Tools. Diese frei verfügbare Tool-Suite unterstützt digitale wissenschaftliche Workflows für die kollaborative Erstellung und Veröffentlichung wissenschaftlicher und dokumentarischer Editionen sowie Sammlungen im Web – ganz ohne Softwareinstallation. Die LEAF Commons Suite ermöglicht die Nutzung einzelner Tools für spezifische Zwecke oder unterstützt einen durchgängigen Workflow von der Transkription bis zur Veröffentlichung und eignet sich somit für ein breites Spektrum an Forschungs- und Lehranwendungen.

[LEAF](https://www.leaf-vre.org/) steht für Linked Editing Academic Framework, eine kollaborative Software-Suite mit modularen Tools für Textbearbeitung und -veröffentlichung. Die LEAF Commons-Tools bieten Forschenden eine leicht zugängliche, kostengünstige Infrastruktur zur Erstellung von Online-Texten, Editionen oder Sammlungen – sei es für Lehre, Forschung oder nachhaltige Zusammenarbeit. Die LEAF Commons-Tools sind im Browser frei verfügbar und ermöglichen die Zusammenarbeit und Veröffentlichung über GitHub sowie die lokale Speicherung oder Installation in einem lokalen System. Die LEAF Commons-Tools fördern die Wiederverwendung von Datenmethoden gemäß den FAIR-Prinzipien (Findable, Accessible, Interoperable, and Reusable). LEAF nutzt Open-Source-Software, Open-Access-Plattformen und offene internationale Standards für Best Practices in der Textkodierung (TEI-XML) und Web-Annotation (RDF).

## LEAF-Writer

LEAF-Writer ist ein Open-Source- und Open-Access-XML-Editor (Extensible Markup Language), der in einem Webbrowser läuft und Wissenschaftlern und Studierenden umfassende Textbearbeitungsmöglichkeiten bietet. Proprietäre Software muss weder heruntergeladen, installiert noch konfiguriert werden, es fallen keine laufenden Abonnementgebühren an und komplexe Programmiersprachen müssen nicht erlernt werden. Diese benutzerfreundliche Bearbeitungsumgebung unterstützt die Standards der Text Encoding Initiative (TEI) und des Resource Description Framework (RDF). Die Anbindung an Linked Open Data Identifiers und projektspezifische Authentizitäten ist Teil des Kodierungsprozesses. LEAF-Writer ist besonders im pädagogischen Kontext wertvoll, da Dozierende Studierenden bewährte Kodierungspraktiken vermitteln können, ohne ihnen Programmierkenntnisse beibringen oder die Installation von Software vor Ort erfordern zu müssen.

### Hauptmerkmale

- Schemabasierte Dokumentenbearbeitung inklusive Validierung anhand webbasierter Schemas
- Unterstützung von Cascading Stylesheets (CSS) für eine WYSIWYG-Ansicht von Dokumenten sowie eine Ansicht mit Tags
- Möglichkeit, Verweise auf benannte Entitäten (Personen, Orte oder Organisationen) aus bereits getaggten XML-Referenzen innerhalb eines Dokuments zu extrahieren, um Web-Annotationen zu generieren
- Möglichkeit, Kennungen für Tags benannter Entitäten (Personen, Organisationen, Orte oder Titel) aus den folgenden Linked Open Data-Datenbanken zu suchen und auszuwählen: [DBPedia](https://dbpedia.org/), [Geonames](https://www.geonames.org/), [Getty](https://www.getty.edu/), [GND](https://www.dnb.de/EN/Professionell/Standardisierung/GND/gnd_node.html), [LINCS Project](https://lincsproject.ca), [VIAF](https://www.viaf.org/) und [Wikidata](https://www.wikidata.org/) oder projektspezifische Normdateien.
- Generierung von Linked-Data-Annotationen für neu getaggte benannte Entitäten und Dokumentannotationen (Datum, Anmerkungen, Zitate, Korrekturen, Links, Schlüsselwörter) in XML-RDF oder JSON-LD gemäß dem Web Annotation Data Model.
- Kontinuierliche XML-Validierung.
- Schema-basierte Tagging-Optionen.

### TEI-Unterstützung

LEAF-Writer ist für die Verwendung mit [Anpassungen des Text Encoding Initiative (TEI)](https://tei-c.org/guidelines/customization/#section-1)-Schemas des TEI-Konsortiums konzipiert. LEAF-Writer unterstützt standardmäßig die folgenden Schemas: [TEI All](https://www.tei-c.org/release/xml/tei/custom/schema/relaxng/tei_all.rng), [TEI LITE](https://www.tei-c.org/release/xml/tei/custom/schema/relaxng/tei_lite.rng), [TEI Simple Print](https://www.tei-c.org/release/xml/tei/custom/schema/relaxng/tei_simplePrint.rng), [jTEI Article](https://www.tei-c.org/release/xml/tei/custom/schema/relaxng/tei_jtei.rng) und [Orlando](https://cwrc.ca/schemas/orlando_entry.rng).

### Benutzerdefinierte Schemas

LEAF-Writer unterstützt auch benutzerdefinierte Schemas. Beim Öffnen eines Dokuments prüft LEAF-Writer das Wurzelelement und die Schemadefinition. Aktuell unterstützt LEAF-Writer drei verschiedene Wurzelelemente: `TEI`, `ORLANDO` und `CWRCENTRY`. Wird das Wurzelelement vom Schema nicht unterstützt, können Sie es als benutzerdefiniertes Schema hinzufügen. LEAF-Writer speichert die Schemainformationen im lokalen Speicher des Browsers. Das Schema ist dann lokal verfügbar, solange Sie angemeldet sind.

## Dokumentenspeicherung

LEAF-Writer Commons nutzt GitHub für die Dokumentenspeicherung, Versionsverwaltung und Freigabe. Um diese Funktionen nutzen zu können, müssen Sie in einem GitHub-Konto angemeldet sein. Zusätzlich können Sie Dokumente öffnen, indem Sie eine XML-Datei einfügen oder eine Datei von Ihrem Computer hochladen. Sie können die Datei auch direkt auf Ihr Gerät herunterladen. Optional können Sie LEAF-Writer auch ohne externes Konto verwenden. In diesem Fall können Sie jedoch nur Dateien von Ihrem Computer laden und dort speichern.

Mit LEAF-Writer können Sie XML-Dokumente bearbeiten oder neue Dokumente anhand von Vorlagen erstellen. Vorlagen und Beispieldokumente für den Einstieg finden Sie hier.
